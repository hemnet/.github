Here's a handy reference on how a Pull Request on Hemnet should be written:
https://github.com/hemnet/documentation/wiki/Pull-Request-guide

**The essentials - please remember to:**

* Mention other steps in this PR series, if this is part of a series.
* Write a description of *why* (and optionally *how*) including:
  * Links, PR references, review tips, pairing partner, TODOs.
  * Any other relevant information.
* Mention related teams and assign someone to review.
* Add appropriate labels and a milestone if applicable.

**Also please remember to:**

* Write in English.
* Write the PR so that it is understandable for other teams and future colleagues.
* Include screenshots when possible.
* Quote important content from linked sources, since links might change.
